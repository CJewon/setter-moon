## 회의 배경

랜딩 본문 섹션에 `소개`, `기능`, `흐름` 같은 라벨이 반복되어 상단 내비게이션 문구와 중복되고, 화면이 서비스를 판매하는 랜딩이라기보다 문서 목차처럼 보인다는 피드백이 있었습니다.

사용자 의견:

- `소개`는 별도 본문 라벨보다 Hero 영역의 `주문과 재고를 한 화면에서 가볍게` 메시지가 담당하는 것이 자연스럽다.
- `혼자 운영해도 주문과 재고가 헷갈리지 않게 정리합니다.` 이후 문제 제기 카드는 소개가 아니라 기능 필요성을 설명하는 영역에 가깝다.
- `상품·옵션 조합`, `가용 재고`, `수동 주문 등록`, `운영 대시보드`는 기능 섹션으로 명확히 묶는 것이 좋다.
- 소개와 기능 영역의 UI가 현재 너무 작아 보여 정보 위계와 여백 조정이 필요하다.

## 5명 회의 결론

### 총괄기획자

사용자 의견에 동의합니다. 랜딩에서 `소개/기능/흐름` 같은 라벨을 본문 eyebrow로 반복하면 정보 구조를 설명하는 느낌이 강해지고, 제품 메시지가 약해집니다. Hero를 서비스 소개 영역으로 보고, 본문은 문제 상황, 핵심 기능, 운영 흐름 순으로 자연스럽게 연결합니다.

### 디자이너

본문 라벨은 제거하고 섹션 제목과 설명만으로도 목적이 보이도록 합니다. 소개와 기능 영역은 카드 수만 늘리는 방식이 아니라 제목 크기, 여백, 카드 배치, 미리보기 영역 비중을 조정해 더 넓고 명확하게 보여야 합니다.

### 풀스택개발자

`landing-sections.tsx`와 `landing-content.ts` 기준으로 수정합니다. 현재 랜딩 콘텐츠는 데이터와 섹션 컴포넌트로 분리되어 있어, 문구와 섹션 구조를 비교적 안전하게 조정할 수 있습니다.

### 리뷰어

상단 내비게이션 라벨과 본문 라벨이 중복되는 문제, 문서식 설명처럼 보이는 문제, 모바일에서 섹션 흐름이 답답해지는 문제를 중점 리뷰합니다.

### 테스터

랜딩 화면을 375px, 768px, 1200px 이상 기준으로 확인합니다. 상단 내비게이션 이동, CTA 이동, 섹션 스크롤 위치, 본문 라벨 제거 여부, 소개/기능 영역 가독성을 실제 Chrome 기준으로 검증합니다.

## 수정 정보구조

```txt
Hero
- 서비스 소개 역할
- 주문과 재고를 한 화면에서 가볍게 관리한다는 핵심 메시지
- 무료 시작 CTA
- 화면 미리보기 CTA

문제 상황
- 혼자 운영하며 자주 헷갈리는 주문과 재고 상황
- 옵션이 늘어나며 생기는 재고 혼선
- 주문접수와 배송대기 사이의 재고 숫자 혼선
- 여러 채널 주문을 다시 옮겨 적는 부담

핵심 기능
- 상품·옵션 조합
- 가용 재고
- 수동 주문 등록
- 운영 대시보드

운영 흐름
- 상품 등록
- 옵션 조합 생성
- 초기 재고 입력
- 주문 수동 등록
- 배송대기 전환
- 재고 차감과 이력 확인

요금제와 CTA
- 무료 플랜
- 유료 풀버전 후보
- 무료 시작 CTA
```
